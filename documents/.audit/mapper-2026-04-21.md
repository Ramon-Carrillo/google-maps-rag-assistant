## Deprecations & renames spotted

Naming/currency issues where local docs lag Google's current pages:

1. **Routes API pricing tier names are stale.** Local `billing-and-pricing.md` and `routes-api.md` use "Basic / Advanced / Preferred" tiers. **Google's current pricing page uses "Essentials / Pro / Enterprise"** (exact SKU text: "Routes: Compute Routes Essentials", "Routes: Compute Routes Pro", "Routes: Compute Routes Enterprise", and same triple for Compute Route Matrix).
2. **Places pricing tier names are stale.** Local `billing-and-pricing.md` uses "Place Details (Basic/Contact/Atmosphere)" and "Find Place / Nearby Search / Text Search" flat SKUs. **Google's current pricing uses "Essentials / Pro / Enterprise" tiers** (e.g. "Places API Place Details Essentials", "Places API Place Details Pro", "Places API Place Details Enterprise", "Places API Place Details Enterprise + Atmosphere", "Places API Nearby Search Pro", "Places API Text Search Enterprise", etc.). "Place Photo" is now "Places API Place Details Photos".
3. **Address Validation also uses Pro/Enterprise tiers.** Two SKUs: "Address Validation Pro" and "Address Validation Enterprise". Not covered locally at all.
4. **Places API naming.** Page title is now just "Places API"; "(New)" is used contextually to distinguish from "Places API (Legacy)". Local doc title "Places API (New) and Autocomplete" is acceptable but slightly off-label.
5. **Drawing Library + Heatmap Layer deprecations (missed).** Google's deprecations page lists Drawing Library (deprecated 2025-08-08, sunset May 2026, replacement: Terra Draw) and Heatmap Layer (deprecated 2025-05-27, sunset May 2026, replacement: deck.gl HeatmapLayer). Local `deprecations-2026.md` does not mention either — both are sunsetting within ~1 month of the fetch date.
6. **"Places UI Kit"** is a currently-marketed distinct surface with its own SKUs — our docs never use the phrase.
7. **"Maps Grounding Lite"** appears as a Places-family SKU on the pricing page — name suggests an AI/grounding integration; needs manual confirm.
