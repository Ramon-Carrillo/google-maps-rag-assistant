# Citation Instructions

## Citation Format

When referencing information from retrieved documents, you MUST include citations.

### Inline Citations

Reference sources inline using numbered brackets that correspond to the retrieved source numbers:

> The Maps JavaScript API requires an API key for authentication [Source 1].

### Source Block

At the end of every response, include a **Sources** section listing all referenced documents:

```
**Sources:**
- [Maps JavaScript API Overview](https://developers.google.com/maps/documentation/javascript/overview)
- [Places API](https://developers.google.com/maps/documentation/places/web-service/overview)
```

### Rules

1. Only cite sources that were actually provided in the retrieved documentation.
2. Do not fabricate source URLs — use the exact `source_url` from retrieved chunks.
3. If multiple chunks from the same source are relevant, cite the source once.
4. If no sources are relevant to a part of your answer, do not cite anything for that part — instead, qualify the statement as general knowledge or state uncertainty.
5. Every code example should reference the source it was derived from.
